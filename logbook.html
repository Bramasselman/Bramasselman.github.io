Some intermediate steps

Initial flights dataframe

Processing all given data in Python (using Pandas). With the provided data being in a rather tedious format, this step took already a lot of time. The result is that all required data is now available in Python in a user-friendly way.


Retrieve Broken Itineraries

Broken itineraries were retrieved based on the criteria listed in the assignment. The passengers part of these itineraries were sorted based on the potential downgrading cost (a passenger flying first class can lead to a large penalty if he/she is downgraded to economy class). This is why based on this criteria the passengers were sorted before adding them to the shortest path loop. More explanation on the shortest path algorithm and implementation is given in Shortest Path.


Remaining Empty Seats for each Flight

After having defined the broken itineraries (and thus passengers that need to be assigned to another (set of) flight(s), the amount of passengers that are part of the non-broken itineraries can be found. This amount is then subtracted from the capacity per flight. This way, the amount of remaining seats for each flight number was obtained. This number will be particularly useful when assigning passengers form broken itineraries to new flights.


Shortest path algorithm

For passengers with cancelled itineraries, starting with passengers inducing highest costs, a shortest path was set-up with the network origin being their departure airport and the network sink being their arrival airport. Along all existing flights with available seating capacity, a path with minimum cost was found and passengers were assigned to those flights. This process was repeated until all passengers were assigned flights, making sure that all passengers would end up at the desired destination.
